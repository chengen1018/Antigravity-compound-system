# 優先級報告格式說明

本目錄存放專案的優先級報告。`auto-compound.ps1` 會自動讀取最新的報告（按檔名日期排序），
從中選出第一優先任務執行。

## 檔名格式

```
priority-report-YYYY-MM-DD.md
```

## 報告結構（必須遵循此格式以便機器解析）

```markdown
# Priority Report YYYY-MM-DD

## Priority Items

### 1. [HIGH] 任務標題
- **描述**: 簡短描述這個任務
- **範圍**: 影響的檔案或模組
- **驗收標準**: 完成的具體條件

### 2. [MEDIUM] 任務標題
- **描述**: ...
- **範圍**: ...
- **驗收標準**: ...

### 3. [LOW] 任務標題
- **描述**: ...
- **範圍**: ...
- **驗收標準**: ...
```

## 注意事項

- `auto-compound.ps1` 只會執行排名第 1 的任務
- 優先級標記 `[HIGH]`/`[MEDIUM]`/`[LOW]` 用於 PR 風險分級
- 每個任務的「驗收標準」會被 Gemini CLI 用來判斷任務是否完成
